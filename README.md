# Roco Kingdom Egg Hatch Predictor

Predict which pokemon will hatch from a Roco Kingdom egg based on its size (M) and weight (KG).

## Quick Start

```bash
# 1. Install dependencies
make install

# 2. Scrape pokemon data from the wiki (one-time setup)
python3 scraper.py

# 3. Launch the web dashboard
make dashboard
# → open http://127.0.0.1:5000 in your browser
```

---

## How It Works

### The Egg-to-Pokemon Size Relationship

When a pokemon hatches, its size and weight are related to the egg's measurements by a multiplier, with a ±0.03 M/KG leniency applied to both edges:

```
pokemon_size   ≈ egg_size   × [2.2 … 3.2]  (−0.03 lower, +0.15 upper tolerance)
pokemon_weight ≈ egg_weight × [1.9 … 3.2]  (−0.03 lower, +0.15 upper tolerance)
```

So given an egg with size `S` and weight `W`, the effective search range is:
- **Size range:** `[S×2.2 − 0.03,  S×3.2 + 0.15]`
- **Weight range:** `[W×1.9 − 0.03, W×3.2 + 0.15]`

### Prediction Algorithm

Each candidate pokemon is scored on how well its known stat ranges overlap with the expected ranges.

**Step 1 — Overlap ratio (60% weight)**

For each dimension (size, weight):
```
overlap_length = max(0, min(exp_max, poke_max) - max(exp_min, poke_min))
ratio = overlap_length / (exp_max - exp_min)
```

**Step 2 — Gaussian fit (40% weight)**

How well does the expected center (`egg_size × 2.7` / `egg_weight × 2.55`) land inside the pokemon's range?
```
center = (poke_min + poke_max) / 2
sigma  = (poke_max - poke_min) × 0.5
score  = exp( -0.5 × ((exp_center - center) / sigma)² )
```
If the center is already inside the range, it still decays toward the edges.

**Step 3 — Combined dimension score**
```
dim_score = 0.6 × overlap_ratio + 0.4 × gaussian_fit
```

**Step 4 — Both-dimension requirement**

| Overlap state               | Base score formula                     |
|-----------------------------|----------------------------------------|
| Both size & weight overlap  | `(size_score + weight_score) / 2`      |
| Only size overlaps          | `size_score × 0.3`  (heavy penalty)    |
| Only weight overlaps        | `weight_score × 0.3`                   |
| Neither overlaps            | excluded                               |

**Step 5 — Bayesian boost from confirmed observations**

When you record a confirmed hatch (egg → pokemon), future predictions for similar eggs get a boost:
- **Similarity window:** ±15% of the input egg's size/weight
- **Boost strength:** proportional to how close the recorded egg was to the current one
- **Formula:** `final_score = base_score × (1 + accumulated_boost)`

**Step 6 — Normalise to percentage**
```
probability(pokemon) = score(pokemon) / sum(all scores) × 100
```

Results are sorted by probability, top 10 shown.

## If the Scraper Gets Blocked (HTTP 567)

biligame.com uses Cloudflare rate-limiting. If `python3 scraper.py` fails for some pokemon, run them individually with a built-in human-like delay:

```bash
python3 scrape_single.py 喵喵
python3 scrape_single.py 石肤蜥
# etc.
```

Each run waits 8–20 seconds before fetching, then merges the result into `data/pokemon.json`.

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/` | Web UI |
| `POST` | `/predict` | `{size, weight}` → ranked predictions |
| `POST` | `/confirm` | `{size, weight, pokemon}` → record a hatch |
| `GET` | `/observations` | List recorded hatches (last 50) |
| `DELETE` | `/observations/<id>` | Delete a hatch record |
| `GET` | `/pokemon` | List all hatchable pokemon |
| `GET` | `/status` | Data loading status |
